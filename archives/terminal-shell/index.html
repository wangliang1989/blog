终端模拟器 和 shell

终端模拟器

终端（英语：Computer terminal），是一台电子计算机或者计算机系统，用来让用户输入数据，及显示其计算结果的机器。 终端其实是一种输入输出设备，相对于计算机主机而言属于外设，本身并不提供运算处理功能。 例如，食堂打饭刷卡的那个机器就是一个终端。它本身并不是一台计算机。 但是，可以靠这个终端输入你的一卡通信息到远程计算机，然后返回信息给终端，终端会对你的一卡通余额做修改或者发出鸣音报警你没钱了。

早期的计算机终端一般是机电的电传打字机。 现代的操作系统如 Linux 和 BSD 上，使用 X Window 这样的图形用户界面时，屏幕上一般有多个与不同应用相关的窗口开着，而不是只有一个与一个单个过程相连的文字流。 这种情况下，用户一般使用终端模拟器，又叫终端模拟程序或者虚拟终端。 这样，用户可以不必使用专门的终端装置来与计算机交换。

shell

shell，中文称为壳层，是指为用户提供用户界面的软件，是操作系统内核 (core) 和用户的中介。 普通意义上的 shell 是接受用户输入命令的程序。 它之所以被称作 shell 是因为它隐藏了操作系统低层的细节。 通常将 shell 分为两类：图形界面与命令行。 图形 shell 提供一个图形用户界面（GUI），命令行shell提供一个命令行界面（CLI）。 图形用户界面 GNOME 和KDE就是图形 shell。 这里要介绍的是命令行 shell。

为什么要更改系统默认终端模拟器和 shell

终端模拟器就是一个虚拟的输入输出设备，好比软键盘。shell 是操作系统的一层。 好比刷一卡通，食堂里打饭刷卡的是一种终端，图书馆闸机是一种终端，自助借书机是一种终端等等。 终端模拟器也有很多种。同样，shell 也是。 GNOME 界面默认的分别是是 GNOME-terminal 和 bash。 bash 也是绝大多数 GNU/Linux 系统和 Mac 上默认的 shell。

我们现在要把这二者改为 GNOME-terminator 和 zsh。 理由是:

  • GNOME-terminator 可以实现分割，这是极为重要的功能
  • zsh 的语法几乎完全兼容于 bash
  • zsh 的命令补全比 bash 的更好用
  • zsh 配置方便

zsh

zsh 和 oh my zsh 的安装

安装 zsh：

sudo yum install zsh

oh my zsh 是一帮人一起维护的一套 zsh 的配置文件。 稍后我们借助它来配置 zsh。

安装 oh my zsh：

curl -L http://install.ohmyz.sh | sh

这条命令，做三件事：

  • 下载 oh my zsh 到 ~/.oh-my-zsh.
  • 备份已有的zsh配置文件 ~/.zshrc ，并复制新的 .zshrc 文件
  • 将当前用户的默认 shell 由 bash 改成 zsh

第三步会报错： chsh: "/usr/bin/zsh" is not listed in /etc/shells. ，需要手动修改默认shell，在终端执行：

chsh -s /bin/zsh

chsh 命令修改的是 login shell，当前用户重新登陆后用户的默认 shell 就从 bash 变成了 zsh。 注意今后环境变量不要写到 .bashrc 而要写到 .zshrc 中。在写环境变量时，一定注意自己添加的内容写到后面去。

用 oh-my-zsh 配置 zsh

zsh 和 bash 一样，有专门的配置环境变量的文件： ~/.zshrc 。 zsh 所有的配置都保存在其中。 在 .zshrc 中可以选择喜欢的主题，以及适当数量的插件。 插件太多，机器会变慢

假设你的安装过程和上面一样，oh-my-zsh 这一套配置文件是放在当前计算机用户的主目录下的，是一个隐藏文件夹。 在 .zshrc 中有一句话来指定这个位置：

export ZSH=/home/计算机用户名/.oh-my-zsh

你可以根据需要改变这个位置。

配置主题 ++++++++++++++

在\ .zshrc\ 中有一句::

ZSH_THEME="robbyrussell"

这句话指定主题是\ robbyrussell\ 。这也是zsh的默认主题。改变这个值就可以改变主题。 zsh的所有主题保存在\ /themes\ 目录下。 你可以看看\ oh my zsh的themes wiki <https://github.com/robbyrussell/oh-my-zsh/wiki/themes>\ 预览各个主题的效果。 我之前用的主题是\ amuse\ 。为了效果理想，你应该安装这个字体：\ Powerline fonts <https://codeload.github.com/PeterPanwl/fonts/zip/master>\ 。 我现在用的是\ bullet-train\。这个需要单独安装。

配置插件 +++++++++ 有一些插件可以选用。

在\ .zshrc\ 中有一句::

plugins=(git)

也许这句话在你那里被注释了。这句话是说使用插件git，你可以仿照着，添加其他插件。下面介绍一些插件：

  1. git

为git的众多常用命令提供名。比如\ git status\ 的别名是\ gst\ ，大大减小了击键数。但该插件中\ git mergetool --no-prompt\ 的别名是\ gmt\ ，与 GMT 软件冲突，需要将该插件的目录 git 复制到 custom/plugins 下，然后删除其中的 gmt 别名；

  1. 命令补全插件: pip, pyenv；
  2. sudo：按两下\ ESC\ 即可在当前命令前加上\ sudo\；
  3. yum：为常见的yum命令提供别名。

terminator

安装terminator::

sudo yum install terminator

在配置 terminator 时，注意全局里的 Appearance 的 Unfocsed terminal font brightness 的如果太小，非当前窗口将回变成黑白。 另外可以考虑不选配置里的一般设定的显示标题栏，可以节约一点空间。
